## 2024-05-24 - Accessible Icon Buttons and Custom Radios in Minesweeper
**Learning:** Icon-only buttons (like music, sound, cascade) and custom radio buttons in this application frequently lack native focus rings and descriptive labels. Relying solely on SVG icons makes them invisible to screen readers, and default browser focus rings are often suppressed or insufficient. Custom radio inputs hiding the native radio with `sr-only` require the parent label to handle focus styles using the `has-[:focus-visible]` modifier.
**Action:** When creating or updating icon-only interactive elements, always include an `aria-label`, a `title` attribute for mouse hover tooltips, and explicit `focus-visible:ring-2 focus-visible:ring-indigo-400 focus-visible:outline-none` Tailwind utility classes to ensure both screen reader and keyboard accessibility. For custom hidden inputs wrapped in labels, use `has-[:focus-visible]` on the label.
