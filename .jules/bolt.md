## 2024-06-04 - Cache trigonometric values for high-frequency physics loops
**Learning:** In the regatta simulation, `getWindAt(x, y)` was called hundreds of times per frame (for every boat, particle, and wave grid cell). Inside, it computed multiple trigonometric functions (`Math.sin`, `Math.cos`) and divisions for the global wind direction and for every active gust/lull. Re-calculating these values per-query for grid-based/particle systems is a major performance bottleneck in JavaScript.
**Action:** When properties like wind direction, gust rotation, or bounding box radii update (e.g., once per frame in `updateBaseWind` or `updateGusts`), pre-compute and cache the required trigonometric values (`sinRot`, `cosRot`) and divisions (e.g., `invRadiusXSq = 1 / (rX*rX)`). Then, in high-frequency spatial queries like `getWindAt`, rely exclusively on these cached values to replace expensive Math function calls and divisions with simple multiplications.
