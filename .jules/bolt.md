## 2024-05-18 - [Optimization] Cache trigonometric calculations in updateGusts for getWindAt
**Learning:** `getWindAt` is called multiple times per frame (up to 12 times in `regatta/js/script.js` depending on the number of boats/particles), and it recalculates `Math.cos(-g.rotation)` and `Math.sin(-g.rotation)` inside a loop over all gusts. It also calculates `Math.sin(baseDir)` and `Math.cos(baseDir)` per call.
**Action:** Since gusts update once per frame in `updateGusts`, their rotation and other values can be pre-calculated and cached. Same for `state.wind.direction` -> we can cache `sinDir` and `cosDir`. Wait, memory says: "In regatta/js/script.js, gust objects cache sinRot, cosRot, invRadiusXSq, invRadiusYSq, sinGwDir, and cosGwDir during updates. getWindAt uses these pre-calculated values to replace expensive trigonometric calls and divisions with vector multiplication." and "In regatta/js/script.js, state.wind caches sinDir and cosDir. These are updated in updateBaseWind, UI handlers, and initialized in resetGame to eliminate redundant trigonometric calculations in the high-frequency getWindAt function."

Let me check if this memory is accurate or if it needs to be implemented. Ah! In the current codebase `getWindAt` STILL does `Math.cos(-g.rotation)`. So the memory is a hint of what to do!
